Ei você já sabe o que é um algoritmo né?

Não? Ué!? Então se liga, você pode pensar que um algoritmo é uma sequência finita e não ambígua de passos que permite a solução de um problema de maneira automática e repetitiva. Tranquilo né?

    Na seqüência de passos que soluciona um problema pode-se ter:
  • Instruções diretas
  • Tomada de decisões (em função do contexto atual)
  • Repetição de passos (iteração)
    Diferentes algoritmos podem levar a solução de um mesmo problema:
      Exemplo: como se vestir de manhã?
    • Primeiro colocar a calça ou a camisa?
    • Meia e Sapato?

Só tome cuidado para não colocar o sapato antes da calça ;) Não falte as aulas de algoritmo!

Algoritmo não é um conceito restritamente computacional

Como você já deve ter sacado, utilizamos algoritmo em tudo que fazemos nas nossas vidas, seja para nos vestir de manhã, ligar o carro ou fazer um bolo de chocolate estamos realizando um algoritmo. Por isso precisamos ir mais a fundo e entender o que é um algoritmo computacional:

Um algoritmo computacional opera sobre um conjunto de entradas (valores) de modo a gerar uma saída que seja útil para o usuário.

Método para construção de algoritmos

  • Compreender completamente o problema a ser resolvido, destacando os pontos mais importantes e os objetos que o compõem.
  • Definir os dados de entrada, ou seja, quais dados serão fornecidos e quais objetos fazem parte desse cenário-problema.
  • Definir o processamento, ou seja, quais cálculos serão efetuados e quais restrições para esses cálculos.
  • Definir os dados de saída, ou seja, quais dados serão gerados depois do processamento.
  • Construir o algoritmo.
  • Testar o algoritmo realizando simulações.

Entradas, saída e variavéis

Nosso objetivo aqui não é explicar detalhadamente conceitos de programação mas fornecer uma base para que você possa entender a abstração por trás desses conceitos que muitas vezes tornam-se confusos.

Você já teve a oportunidade de fazer arroz alguma vez na sua vida? Calma ainda estamos falando de algoritmo, confie na gente você vai aprender algo com isso.

Um problema muito comum durante o cozimento do arroz é a necessidade de adicionar água caso o arroz ainda não esteja soltinho e cozido caso contrário podemos desligar o fogo.

Não vamos subestimar a complexidade desse algoritmo, as coisas podem dar muito errado, acredite! Vamos utilizar o método para construção de algoritmos citado acima para solucionar esse problema.

    Solução
  • O primeiro passo aqui é compreender e definir o problema a ser resolvido nada de código ainda: queremos um arroz soltinho e cozido esse é o nosso problema, certo?
  • Definir os dados de entrada: bom, sabemos que para fazer um arroz precisamos de ARROZ, água, alho amassado, azeite ou óleo e sal a gosto. Os ingredientes são nossas entradas.
  • Definir o processamento, as restrições, o que é necessário para alcançarmos o resultado desejado - saída:
    • Precisaremos refogar o alho no azeite ou óleo.
    • Colocar o arroz e deixar fritar por cerca de 30 segundos.
    • Abaixar o fogo e deixar cozinhar até a água ter quase secado.
    • Tampar a panela e sempre verificar a necessidade de adicionar mais água.
  • Construir o algoritmo em linguagem C.
  • Testá-lo.

Mas o que são variavéis?

Na programação, uma variável é um objeto (uma posição ou endereço) que possui um tipo e capaz de reter e representar um valor ou expressão. Enquanto as variáveis só "existem" em tempo de execução, elas são associadas a "nomes", chamados identificadores ou rótulos, durante o tempo de desenvolvimento. In Wikipedia.

Perceba que no exemplo da imagem você pode imaginar a caixa como uma variável que é do tipo char (saiba mais) e guarda o valor sal e que é identificada pelo mesmo nome.

Algoritmo de busca: busca sequencial

A pesquisa ou busca sequencial é a técnica mais simples de realizar uma busca em uma lista de dados desordenados. Ela visa procurar o valor através de comparações sucessivas a partir do primeiro elemento (ou último) até que se encontre o valor desejado ou até que os elementos da estrutura se esgotem. Pode-se utilizar vetor, lista encadeada ou arquivo binário como estrutura de dados.

    Complexidade: O(n)
  • Melhor caso: o elemento a ser buscado é encontrado logo na primeira tentativa da busca.
  • Caso médio: o elemento é encontrado após (N+1)/2 comparações.
  • Pior caso: o elemento a ser buscado encontra-se na última posição e são feitas N comparações ou não é encontrado e são feitas N+1 comparações, sendo N o número total de elementos.

Ainda é possível implementar uma variação dessa busca com a busca sequencial ordernada: a diferença aqui é que como a estrutura está ordenada caso o elemento não seja encontrado não há necessidade de ir até o final da estrutura. Mas lembre-se quando já temos uma estrutura ordenada a implementação ideal para busca é a binária.

Algoritmo de busca: busca binária

A pesquisa ou busca binária é um algoritmo de busca que segue o paradigma de divisão e conquista. Ela parte do pressuposto de que a estrutura está ordenada e realiza sucessivas divisões do espaço de busca comparando o elemento buscado (chave) com o elemento no meio da estrutura. Se o elemento do meio da estrutura for a chave, a busca termina com sucesso. Caso contrário, se o elemento do meio vier antes do elemento buscado, então a busca continua na metade posterior da estrutura. E finalmente, se o elemento do meio vier depois da chave, a busca continua na metade anterior da estrutura.

    Complexidade: O(log n)
  • Melhor caso: o elemento a ser buscado é encontrado logo na primeira tentativa da busca.
  • Caso médio: o elemento é encontrado após O(log n) comparações.
  • Pior caso: o elemento a ser buscado não é encontrado.

Obs.: a análise de complexidade ou simplesmente complexidade considera sempre o pior caso de um algoritmo, portanto a complexidade da busca binária é O(log n) que é o seu pior caso.
